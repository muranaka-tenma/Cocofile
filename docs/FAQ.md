# よくある質問（FAQ）

CocoFile に関するよくある質問と回答をまとめました。

---

## 📥 インストール・セットアップ

### Q: どのOSに対応していますか？

**A**: Linux、Windows、macOS の全てに対応しています。

- **Linux**: Ubuntu 20.04以降、その他のディストリビューション（AppImage）
- **Windows**: Windows 10 (1803以降)、Windows 11
- **macOS**: macOS 13 (Ventura) 以降

### Q: インストール方法は？

**A**: プラットフォームごとに異なります：

- **Linux**: `.deb`（Debian/Ubuntu）または `AppImage`（全ディストリビューション）
- **Windows**: `.msi` または `.exe` インストーラー
- **macOS**: `.dmg` パッケージ

詳細は[ベータテストガイド](BETA_TEST_GUIDE.md)を参照してください。

### Q: Windows Defender に警告が出ます

**A**: 開発者署名がないため、SmartScreen の警告が表示されます。

**解決方法**:
1. 「詳細情報」をクリック
2. 「実行」をクリック

CocoFileは完全にオープンソースで、マルウェアは含まれていません。

### Q: macOS で「開発元を確認できません」と表示される

**A**: macOS のセキュリティ設定により、未署名のアプリは警告が表示されます。

**解決方法**:
1. アプリを右クリック
2. 「開く」を選択
3. 「開く」をクリック

または、システム設定 → プライバシーとセキュリティ → 「このまま開く」

## 🔍 使い方

### Q: どんなファイルを検索できますか？

**A**: 以下のファイル形式に対応しています：

- **PDF**: .pdf
- **Excel**: .xlsx, .xls
- **Word**: .docx
- **PowerPoint**: .pptx

テキストファイル(.txt)やその他の形式は現在未対応です（将来的に追加予定）。

### Q: 日本語検索は可能ですか？

**A**: はい、完全に対応しています。

CocoFileは N-gram tokenizer を使用しており、日本語の全文検索が可能です。

### Q: 検索速度はどれくらいですか？

**A**: ファイル数によりますが、目標は以下の通りです：

- **小規模（100ファイル未満）**: 即座
- **中規模（1,000ファイル）**: 0.5秒以内
- **大規模（10,000ファイル）**: 1秒以内

実際の速度はハードウェアやファイルサイズに依存します。

### Q: フォルダをスキャンするのにどれくらい時間がかかりますか？

**A**: ファイル数とサイズによりますが、目安は以下の通りです：

- **1,000ファイル**: 数分
- **10,000ファイル**: 10-30分
- **100,000ファイル**: 数時間

スキャンは一度だけで、その後は高速検索が可能です。

### Q: スキャン中にPCが重くなります

**A**: スキャン時のCPU使用率は50%以下を目標にしています。

**対策**:
- 他のアプリを閉じる
- スキャンをバックグラウンドで実行
- 大量のファイルは分割してスキャン

## 🔐 プライバシー・セキュリティ

### Q: データはどこに保存されますか？

**A**: すべてローカルに保存されます。

- **データベース**: ローカルのSQLiteファイル
- **場所**: アプリのデータディレクトリ（OSごとに異なる）
- **外部送信**: 一切なし

### Q: インターネット接続は必要ですか？

**A**: いいえ、完全オフラインで動作します。

CocoFileは外部サーバーとの通信を一切行いません。

### Q: ファイルの内容は暗号化されますか？

**A**: Phase 1では暗号化機能はありません。

将来のバージョンで検討予定です。現時点では、OSのファイルシステム暗号化（BitLocker、FileVault など）を使用してください。

### Q: 機密ファイルをスキャンしても大丈夫ですか？

**A**: データは完全にローカルに保存され、外部に送信されることはありません。

ただし、他のユーザーとPCを共有している場合は注意が必要です。

## ⚙️ パフォーマンス

### Q: メモリ使用量はどれくらいですか？

**A**: 目標値は以下の通りです：

- **アイドル時**: 30-40MB
- **スキャン時**: 500MB以下
- **検索時**: 100-200MB

実際の使用量はスキャンするファイル数やサイズに依存します。

### Q: 大量のファイルをスキャンできますか？

**A**: はい、設計上は10万ファイル以上に対応しています。

ただし、パフォーマンスはハードウェアに依存します。

### Q: 大きなファイル（100MB以上）は処理できますか？

**A**: 可能ですが、時間がかかります。

- **100MB以下**: 通常の速度
- **100MB超**: 警告表示、処理時間が長くなる
- **500MB超**: 非推奨（処理に時間がかかる）

## 🐛 トラブルシューティング

### Q: アプリが起動しません

**A**: 以下を確認してください：

**Linux**:
```bash
# 依存関係を確認
ldd /usr/bin/cocofile

# 必要なパッケージをインストール
sudo apt-get install libwebkit2gtk-4.1-0 libgtk-3-0 libsqlite3-0
```

**Windows**:
- .NET Framework 4.7.2以降がインストールされているか確認
- 管理者権限で実行してみる

**macOS**:
- 「開発元を確認できません」の場合は右クリック → 「開く」

### Q: スキャンが途中で止まります

**A**: 以下の可能性があります：

1. **大きなファイル**: 100MB以上のファイルで処理に時間がかかっている
2. **破損ファイル**: 破損したPDFやOfficeファイルでエラーが発生
3. **メモリ不足**: メモリ使用量を確認

**対策**:
- スキャンログを確認
- 問題のファイルを特定して除外
- メモリを増やす

### Q: 検索結果が表示されません

**A**: 以下を確認してください：

1. **スキャン完了**: フォルダのスキャンが完了しているか
2. **キーワード**: 正しいキーワードを入力しているか
3. **フィルター**: ファイルタイプフィルターを確認

### Q: AppImageが起動しない（Linux）

**A**: FUSE が必要です。

```bash
# FUSEをインストール
sudo apt-get install fuse libfuse2

# または、FUSE不要で実行
./CocoFile_0.1.0_amd64.AppImage --appimage-extract-and-run
```

## 🚀 機能

### Q: 今後追加される機能は？

**A**: Phase 13（v0.2.0）で以下を検討中です：

- ファイルプレビュー機能
- 日付範囲フィルターUI
- 自動スキャン機能
- 正規表現検索

詳細は[Phase 12 計画](PHASE12_PLAN.md)を参照してください。

### Q: OCR（画像内テキスト抽出）に対応していますか？

**A**: Phase 1では未対応です。

将来のバージョンで検討予定です。

**A**: ローカルに同期されたファイルはスキャン可能です。

クラウド上のファイルを直接スキャンする機能は、将来的に検討予定です。

### Q: 複数のPCで同じデータベースを使えますか？

**A**: 現時点では未対応です。

将来的に、データベースのエクスポート/インポート機能を検討予定です。
